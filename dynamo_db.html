Dynamo DB

Dec 2, 2020 #nosql_dbs

Notes regarding Dynamo DB

autoscaling for storage.

bad things

it’s limited by 700 rps per connection. so you should keep multiple connections

it takes about 20 minutes to scale from 1000 write requests to 5000

to decrease number of workers you need to send lower number of write requests. it’s not enough to just stop sending write requests. you can’t change capacity more often than 20 minus.

alternatives:

Redis, Casandra

Mongo doesn’t work if you need autoscaling in terms of storage
