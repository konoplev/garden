Terraform

Oct 19, 2020 #devops

Quick intro to terraform

Idea

You declare desired configuration (set of resources) → terraform reads it and create the requested resources.

For each resource type (s3 bucket for example) there is a provider (a thing that is able to create a requested resource).

terraform init

Command parsing the configuration and loading necessary plugins (providers).

terraform apply

Command reading resources description, creating a dependency tree (how the resources depends on each other) and creating resources starting from the tree’s root.

State

To keep state, terraform writes results of the last terraform apply into a file. A file could be kept locally (default) or shared among team members.

To share the file, it’s needed to create a backed. For example:

locals {
  local_state = {
    backend = "local"
    config = {
      path = var.local_path
    }
  }
  remote_state = {
    backend = "s3"
    config = {
      key      = var.remote_path
      region   = var.remote_region
      bucket   = var.remote_bucket
      role_arn = var.remote_role_arn
    }
  }
}

terraform plan

Same as apply, but it doesn’t change the state (so it creates nothing). It just informs which changes will be done by terraform apply.

terrafrom workspaces

One of the ways to separate environments. You create different workspaces, terraform keep states separately(in env directory)

Another way is to keep different environments in different directories.
